# 视频时序稳定度指标：调研与实施说明

本文说明 Demo 中「原画 / 重绘稳定度、差值、近 5 秒差值均值」指标背后的文献与工程依据，以及在本仓库中的落地方式。

---

## 一、调研了哪些资料与方向

调研主要围绕三类问题：**如何定义「时序一致」**、**视频生成/风格化里常用什么可算指标**、**实时 Demo 能否在 CPU 上跑得动**。

### 1. 基于光流的短时对齐误差（与本实现最直接相关）

- **思路来源**：视频风格化、视频扩散、盲视频增强等领域长期使用「用估计出的运动场把上一帧 warp 到当前帧，再与当前帧做像素/梯度域差异」作为**短时 temporally coherent** 的代理目标；与「相邻帧直接做差」相比，能在有**全局运动**时区分「跟着场景动」与「无运动下的闪烁」。
- **代表性脉络**（概念层面，非逐篇复现）：
  - 基于稠密对应关系（含光流）的时序约束、warping error 最小化，在 ECCV / ICCV 系视频风格化、coherent video processing 工作中反复出现。
  - 工程与综述类表述中常将 **short-term warping error** 与 temporal consistency 写在一起讨论。

### 2. 更强但偏评测 / 训练向的指标（本次未采用）

| 方向 | 简述 | 为何未进本 Demo |
|------|------|-----------------|
| **Fréchet Video Motion Distance（FVMD）等** | 基于运动表征与分布距离的生成视频运动一致性指标（如 [FVMD 论文](https://arxiv.org/html/2407.16124v1)）。 | 需要序列与特征提取管线，偏离线评测，不适合每帧实时调试条。 |
| **GeCo 等可微几何一致性** | 融合运动与结构先验的 warping / 不一致性图（如 [GeCo](https://arxiv.org/html/2512.22274v1)）。 | 偏训练损失或高质量分析，依赖更重，超出「调试面板数值」范围。 |
| **深度光流（如 RAFT）** | 光流更准，遮挡与大幅运动更稳。 | 模型与算力成本高；当前目标为**可部署的实时侧指标**。 |

### 3. 经典稠密光流（本实现选用）

- **OpenCV Farneback**（`calcOpticalFlowFarneback`）：经典多项式展开稠密光流，**无额外深度学习依赖**（在已引入 `opencv-python-headless` 的前提下），适合在 **256 尺度** 等下采样分辨率上控制 CPU 耗时。

结论：**文献与工程共识**支持用「输入上估计光流 → 同一光流对齐前后帧 → L1 类残差」衡量短时一致性；**落地取舍**是 Farneback + 下采样，而非 FVMD/深度光流。

## 二、调研结论（设计决策）

1. **不用裸帧差（MSE/L1 相邻帧）作为唯一指标**  
   大运动会被误判为「不稳定」，与「AI 重绘是否闪烁」不对齐。

2. **采用「输入光流 + 双路 warping 残差」**  
   - 在**连续两帧输入**的灰度图上估计稠密光流 \(F\)（下采样计算，再按比例映射回全分辨率）。  
   - **原画**：\(e_\text{src} = \mathrm{mean}|I_t - \mathrm{warp}(I_{t-1}, F)|\)（灰度，归一化到约 \([0,1]\)）。  
   - **重绘**：用**同一** \(F\) 对上一帧 / 当前帧**输出 RGB** 做 `remap`，  
     \(e_\text{out} = \mathrm{mean}|O_t - \mathrm{warp}(O_{t-1}, F)|\)。  
   - **含义**：在「与输入一致的运动模型」下，重绘仍对不齐 → 更贴近**纹理漂移、闪烁**；原画残差作**同定义基线**（遮挡时两路都可能升高）。

3. **稳定度分数（便于 UI）**  
   \[
   s = \frac{100}{1 + k \cdot e},\quad k=50\ (\texttt{STABILITY\_K})
   \]  
   残差越小，分数越高。

4. **差值与 5 秒滑动平均**  
   - \(\Delta = s_\text{out} - s_\text{src}\)：正表示相对原画更「稳」，负表示重绘相对更抖。  
   - **近 5 秒 \(\Delta\) 均值**：服务端用 `time.monotonic()` 记录每次 \(\Delta\)，维护滑动时间窗，与前端 1 Hz 轮询解耦。

5. **局限（文档化）**  
   - 单进程**全局 Pipeline** 时，多用户会共享时序状态（仅适合单用户 Demo）。  
   - Farneback 在快速运动、遮挡处误差大，数值需结合画面解读。  
   - 未装 OpenCV 时模块跳过计算，API 中对应字段为 `null`，界面显示「—」。

## 三、如何在本项目中落地

### 1. 依赖

- [`Live2Diff/setup.py`](../../setup.py)：`install_requires` 增加 `opencv-python-headless>=4.5,<5`。

### 2. 核心算法模块

- [`demo/temporal_stability.py`](../temporal_stability.py)：`compute_flow_warp_errors(...)`、`stability_score(...)`。  
  - 默认 `flow_max_side=256`。  
  - `import cv2` 失败时返回 `None`，不拖垮推理。

### 3. 与推理管线对接

- [`demo/vid2vid.py`](../vid2vid.py) 内 `Pipeline.predict`：  
  - 在 `has_prepared` 且已有上一帧输入/输出时调用 `compute_flow_warp_errors`。  
  - 输入、输出统一缩放到配置 `width × height` 再比。  
  - `not has_prepared` 时 `_reset_temporal_state()`，清空上一帧与 deque。  
  - `get_debug_info` 合并字段：  
    `temporal_stability_src` / `out` / `delta` / `delta_avg_5s`，以及调试用的 `temporal_stability_e_src` / `e_out`。

### 4. 数据到前端

- 现有 [`demo/app.py`](../app.py) 将 `debug_info` 写入 `ConnectionManager`；`/api/debug/{user_id}` 返回 JSON。  
- [`demo/frontend/src/lib/types.ts`](../frontend/src/lib/types.ts)：`DebugInfo` 增加上述可选字段。  
- [`demo/frontend/src/lib/components/DebugPanel.svelte`](../frontend/src/lib/components/DebugPanel.svelte)：调试区展示  
  **原画稳定度、重绘稳定度、稳定度差值、Δ 近5s 均值**（保留两位小数，`null` 显示「—」）。

### 5. 首帧与 warmup

- **warming_up** 或未积累上一帧时：四项为 `null`。  
- **running** 后从**第二帧**起开始有稳定度与差值；5 秒均值随样本在时间窗内累积。
