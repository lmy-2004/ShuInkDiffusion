# 毕设检查汇报讲义 3.0：实时水墨视频扩散的阶段性实现

## 一、汇报开场

这次检查我主要汇报的是最近一阶段围绕“实时水墨视频扩散稳定性”做的工程推进。前一阶段已经把 `Stable Diffusion 1.5 + 国画 DreamBooth + 水墨 LoRA + Live2Diff` 的主链跑通，系统可以完成实时视频转绘。但跑通以后，问题也变得更具体：水墨效果本身可以出来，但视频里会出现轮廓细抖、白底呼吸、墨纹跳动、主体和背景边界不稳定等问题。

这一阶段没有把工作重点放在继续堆风格模型上，而是围绕实时链路本身做了几类改造：

1. 在原有 `depth` 条件之外，接入 `softedge` 条件，让模型获得更明确的轮廓信息。
2. 对 `depth_scale` 和 `softedge_scale` 做参数热力图，分析结构条件为什么有时反而会带来抖动。
3. 接入光流，把“帧间运动”变成系统内部可用的工程信号。
4. 基于光流做 `warped noise` 和关键步同步，减少每帧重新采样带来的随机闪烁。
5. 用同一套光流建立稳定性指标，把主观的“抖不抖”转成可记录的数值。
6. 加入主体 `mask` 和进 Latent 前的实时 shader，把水墨里的主体保留、背景简化、留白控制分开处理。

所以这次汇报的重点不是说“我又加了几个模块”，而是说明当前系统怎样从一个能跑的实时水墨 demo，逐步变成一个能分析、能调参、能解释问题来源的实时视频扩散系统。

## 二、SoftEdge 接入：为什么在 depth 之外再加一条轮廓条件

最开始系统主要依赖 `depth` 做结构控制。`depth` 的优势很明显，它能给模型提供主体和背景的前后关系，也能约束人物姿态、大块场景结构和空间层次。但水墨视频里有一类问题，单靠 `depth` 不太容易解决：边缘线条会抖，人物轮廓会忽粗忽细，背景留白和物体边界之间会有轻微漂移。

这类问题本质上更接近“轮廓”和“细边”的不稳定，而不是“大结构”的不稳定。因此我在原来的 `depth` 分支之外，接入了一条 `softedge` 分支。它的目的不是替代 `depth`，而是补上 `depth` 对细边信息表达不足的问题。

当前 `softedge` 的提取有两种入口：

1. `classical`：默认使用的轻量方式，主要流程是灰度化、高斯模糊、Sobel 梯度、再次平滑和归一化。
2. `pidinet`：预留的学习型边缘检测入口，如果环境中有 `controlnet_aux` 和对应模型，可以尝试使用。

目前默认使用 classical 方案，是因为它足够轻，不需要额外模型，也更适合实时 demo。它不会像完整边缘检测网络那样增加明显推理负担，便于先验证“轮廓条件能不能接进当前 Live2Diff 主链”。

在工程接入上，`softedge` 没有直接采用完整 ControlNet 方案。完整 ControlNet 的控制方式是在每个 denoising step 额外运行一条控制网络，并把多尺度 residual 注入 UNet 的不同层。这种方案控制力更强，但对当前系统不合适：第一，它会明显增加每帧推理成本；第二，当前项目已经有 warmup UNet、streaming UNet、TensorRT 适配和 wrapper 输入签名，完整 ControlNet 会带来很大的改动面；第三，毕设阶段更需要先做一个实时可跑、可调参、能分析趋势的版本。

所以当前采用的是 early-fusion 的接法。具体过程是：

1. 输入帧先在扩散循环外提取 `softedge_map`。
2. `softedge_map` 和 `depth_map` 一样，通过 VAE encoder 编码到 latent 空间。
3. Streaming UNet 新增 `edge_flow_conv_in`，专门把 `softedge_latent` 映射到和主干入口一致的特征维度。
4. 主图像 latent 经过 `conv_in` 后，`depth` 分支经过 `flow_conv_in`，`softedge` 分支经过 `edge_flow_conv_in`，三者在 UNet 入口处相加。

可以理解为：这不是完整 ControlNet 的“多层控制”，而是在 UNet 最早的入口特征上给模型加一份轮廓提示。它的控制力度比完整 ControlNet 弱，但成本低，和当前实时链路更兼容。

为了让这个分支在没有重新训练的情况下先能工作，我还做了一个初始化处理：`edge_flow_conv_in` 的权重从原来的 `depth` 分支 `flow_conv_in` 拷贝过来。这样新分支不是完全随机初始化，而是先借用一条已经能把条件 latent 映射进 UNet 的通路。这个做法不是最终最优，因为 `depth` 和 `softedge` 的统计分布不一样，但它能让第一版无需训练就先接通。

目前已经完成的接入包括：

1. `pipeline_stream_animation_depth.py` 中的 `build_softedge_map()`、`encode_softedge_map()`、`softedge_latent_buffer`。
2. warmup UNet 和 streaming UNet 中新增 `edge_flow_conv_in`。
3. forward 输入扩展出 `softedge_sample`。
4. wrapper 和 demo 参数中加入 `use_softedge`、`softedge_scale`、`softedge_mode`、`softedge_debug`。
5. demo 端增加 softedge 调试图接口，方便直接观察边缘条件是否正常。
6. TensorRT 模型输入也同步增加 `softedge_sample`，避免 PyTorch 路径和部署路径接口不一致。

这部分工作完成后，系统从 `depth-only` 推进到了 `depth + softedge` 双条件结构控制。`depth` 管大结构，`softedge` 管轮廓线索，这是后续做水墨稳定性分析的基础。

## 三、SoftEdge Control：它现在控制了什么，也暴露了什么

接入 `softedge` 之后，真正要回答的问题不是“有没有边缘图”，而是“这条边缘图进入扩散模型以后到底起什么作用”。从目前观察看，它确实能增强轮廓意识，尤其在人像边缘、衣服轮廓、物体和背景交界处，模型会更倾向于保留明确的边界。这对水墨风格有帮助，因为水墨画面里的线和留白边界非常影响观感。

但当前 `softedge control` 还不是一个成熟的时序稳定模块。原因主要有三个。

第一，`softedge` 本身是高频条件。水墨视频里最容易抖的也恰好是高频部分，比如头发边缘、脸部轮廓、衣服褶皱、背景细线和压缩噪声。`softedge` 如果提取到了这些细节，它会把这些高频变化送进模型；当 `softedge_scale` 增大时，模型会更强地响应这些变化。

第二，当前 `softedge` 是逐帧提取、逐帧归一化。也就是说，每一帧都会根据本帧自己的边缘最小值和最大值重新拉伸强度。这样做在单帧上合理，但在视频里会带来问题：相邻帧即使画面差不多，边缘强度的标尺也可能不同。结果模型看到的不是稳定的一条边，而是每帧被重新定义过强弱关系的边缘条件。

第三，`edge_flow_conv_in` 目前只是从 `depth` 分支复制初始化，没有经过专门的 edge 分布训练。`depth` 是低频、连续、结构性的；`softedge` 是高频、稀疏、尖锐的。两者输入统计差别很大。所以当前 `softedge` 分支能工作，但更准确地说，它还是“借 depth 通路临时跑起来”的状态，不是一个专门训练过的 edge 控制器。

因此我现在对 `softedge control` 的阶段判断是：它已经证明了轮廓条件能接入当前实时扩散链路，也能增强画面边缘表达；但它目前更像轮廓增强器，还不能直接当作时序稳定器。后续如果要继续提高 `softedge` 的权重，必须先处理它的时序归一化和平滑问题，否则调大以后很可能同时放大边缘抖动。

## 四、Depth / SoftEdge 热力图：为什么 edge 变强后稳定性反而变差

为了避免只凭主观观察，我对 `depth_scale` 和 `softedge_scale` 做了参数热力图。热力图横轴是 `softedge_scale`，纵轴是 `depth_scale`，格子里的分数来自 `temporal_stability_delta_avg_30s`，也就是最近 30 秒内“重绘稳定度减去原画稳定度”的平均值。

这个分数要注意理解。它不是输出视频的绝对质量分，而是输出相对于原输入视频额外引入了多少不稳定。如果分数为负，说明模型输出比原视频更抖；越负，说明额外抖动越明显。

从热力图看到的主趋势是：

1. `softedge_scale` 从低到高增大时，稳定性分数整体变差。
2. `depth_scale` 在某些低 `softedge` 区域有补偿作用，但趋势不强。
3. 当 `softedge_scale` 已经较高时，单纯提高 `depth_scale` 很难把分数拉回来。

这个结果一开始看起来不符合直觉，因为我们容易认为“边缘条件越强，轮廓越稳”。但结合当前实现，它其实是合理的。

`softedge` 放大的不是“稳定轮廓”这个抽象概念，而是当前帧里被提取出来的边缘图。如果边缘图本身在视频里轻微跳动，那么 `softedge_scale` 增大以后，模型收到的就是更强的边缘跳动。尤其水墨风格对轮廓线粗细、留白边缘和墨色边界很敏感，边缘条件一抖，输出画面会比普通风格更容易看出闪烁。

`depth` 的表现则不同。它对主体层次和大结构有帮助，但它不直接表达墨线和细边。热力图里真正影响分数的，很可能是线条粗细、背景纹理和留白边缘的小尺度抖动，这些不是 `depth` 最擅长控制的内容。再加上当前 depth map 自己也是逐帧归一化，深度估计结果也会有轻微尺度漂移，所以 `depth_scale` 的改善会显得有限、不完全单调。

这张热力图给我的结论不是“softedge 没用”，而是：当前 softedge 的使用方式还没把“边缘增强”变成“边缘稳定”。现阶段更稳妥的参数策略是把 `softedge_scale` 控制在较低范围，把它当弱轮廓提示；后续先做条件图的时序稳定化，再考虑提高权重。

同时，这张热力图也暴露了实验流程需要继续改进。当前 benchmark 每个组合读取的是滑动窗口分数，如果组合之间没有显式 reset，上一段视频的缓存、噪声状态、稳定度窗口可能影响下一组参数。所以热力图的大趋势可以参考，但局部单个格子不能过度解释。下一步需要在每组参数前调用 reset，并对每组多次采样取平均。

## 五、光流接入：把视频运动变成系统内部信号

在接入光流之前，系统虽然有输入帧和输出帧，但并不知道两帧之间具体发生了什么运动。对于视频稳定性来说，这个信息很关键。因为相邻帧发生变化有两种情况：一种是画面本身正常运动，比如人物转头、手移动、镜头移动；另一种是模型额外引入的闪烁，比如同一块背景忽明忽暗、墨纹每帧变化。没有运动估计，就很难区分这两类变化。

当前项目使用 Farneback 稠密光流作为默认方案。选择它主要是出于实时和工程成本考虑：它来自 OpenCV，不需要额外深度模型，可以在下采样尺寸上运行，适合作为 demo 里的轻量运动估计模块。它不是最准确的光流，但足够把“低运动、中运动、高运动”的趋势提供给后续控制模块。

光流接入后，每帧会在当前输入帧和上一输入帧之间计算：

1. `flow`：每个像素的二维运动向量。
2. `map_x / map_y`：用于 OpenCV remap 的采样网格。
3. `e_src`：原视频在光流对齐后的残差。
4. `mean_flow_mag`：平均运动幅度。

这里的 `e_src` 非常重要。它不只是指标里的原画残差，也被用作 `motion_score`。如果 `e_src` 很小，说明当前帧和上一帧在光流对齐后差异很小，可以认为是低运动或稳定场景；如果 `e_src` 较大，说明画面变化、遮挡、运动模糊或者光流误差更多，系统在使用历史信息时就应该更保守。

工程上，光流计算放在 demo 层。`vid2vid.py` 在 `has_prepared` 之后，会把当前输入帧缩放到推理尺寸，然后和上一帧输入计算光流。算出的 `flow_stats` 会同时传给几个地方：

1. 噪声控制模块，用于计算 `noise_rate` 和 warp 历史噪声。
2. 关键步同步模块，用于 warp 上一帧 `x0` 记忆。
3. 稳定性指标模块，用同一条光流比较原画和重绘结果。
4. 主体 mask 模块，用光流对上一帧 mask 做对齐和 EMA。

这样设计的好处是，光流不再只是一个评测工具，而是变成了当前系统的时序基础设施。它把输入视频的运动关系送进了扩散采样、噪声控制、mask 平滑和指标计算。

## 六、光流驱动的 Warped Noise：让随机噪声沿着画面运动

在扩散视频里，随机噪声是一个很容易被忽略但影响很大的因素。即使输入画面不动，如果每一帧进入扩散时都重新采样一份 fresh noise，模型也可能在同一个位置生成不同的墨纹、不同的白底颗粒和不同的边缘细节。对于普通视频这可能只是轻微纹理变化，但对水墨风格来说会很明显，因为水墨画面里大面积留白和淡墨区域很容易暴露这种随机抖动。

所以我做的第一步不是直接训练时序模型，而是在推理侧控制噪声。A2 运动感知噪声控制的基本思路是：低运动时少引入 fresh noise，高运动时保留更多 fresh noise。

代码里通过 `set_noise_control()` 把每帧的 `motion_score`、`mean_flow_mag` 和 `flow` 传入 pipeline。`_resolve_noise_rate()` 会根据低阈值和高阈值把 `motion_score` 映射到 `[min_noise_rate, max_noise_rate]`。理论上，低运动场景的 `noise_rate` 应该接近下界，高运动场景接近上界。

在这个基础上，我又接了 `warped noise`。它不是简单降低噪声，而是把上一帧缓存下来的噪声用当前光流 warp 到当前帧 latent 网格，再和 fresh noise 混合。流程可以概括为：

1. 在 RGB 空间计算当前帧相对上一帧的光流。
2. 把光流缩放到 latent 尺寸。
3. 用 `grid_sample` 把上一帧的 `prev_noise_latent` 或 `prev_buffer_noise` 对齐到当前帧。
4. 按 `warped_noise_reuse` 和 `warped_noise_residual` 与 fresh noise 混合。

这样做以后，噪声不再每帧完全独立，而是会部分沿着画面运动传播。对于水墨视频，这相当于让墨纹和颗粒的随机扰动有一定连续性，而不是每一帧重新“长”一遍。

目前 demo 配置里已经开启 `enable_warped_noise`，并设置较高的 `warped_noise_blend`。这说明当前实验重点是先验证历史噪声传播对稳定性的帮助。需要说明的是，当前 `min_noise_rate` 和 `max_noise_rate` 还都设为 `1.0`，也就是说动态降噪的接口已经打通，但默认配置还没有真正把低运动时 fresh noise 降下来。后续这部分需要继续调参，尤其要测试静止人像和慢速运动场景，看降低 `min_noise_rate` 是否能进一步减少白底呼吸。

这一块目前的阶段意义是：系统已经不再把噪声当作每帧独立的随机项，而是开始把噪声看作可以沿时间传播、可以随运动调节的变量。这是后续时序稳定的一个重要基础。

## 七、光流驱动的关键步同步：在去噪早期建立跨帧共识

`warped noise` 主要解决的是随机扰动连续性，但它还不能完全解决结构层面的闪烁。比如白底边界每帧轻微变化、主体轮廓粗细跳动、背景墨面位置轻微漂移，这些问题不只是噪声不同，也和扩散模型每帧在去噪过程中对结构的判断不同有关。

因此我又实现了路线二里的一个推理侧版本：无训练递归记忆 + 关键步 `x0` 同步。

这里没有训练新的 temporal adapter，也没有增加可学习 memory。当前做法更轻量：保存上一帧在关键去噪 step 的 `x0` 预测，把它作为递归记忆。下一帧到来时，先用光流把上一帧的 `x0` 对齐到当前帧位置，再和当前帧同一关键 step 的 `x0` 做小权重融合。

核心公式是：

```text
synced_x0 = (1 - weight) * current_x0 + weight * warped_prev_x0
```

这里的 `x0` 可以理解为模型在某个噪声等级下预测出来的“干净 latent”。它比最终 RGB 输出更接近扩散模型内部的结构判断。直接对输出图做后处理容易产生糊和拖影，而在关键步 `x0` 上做轻量同步，可以更早地影响结构生成，同时保留后续去噪步骤的细节修正空间。

为什么选关键步，而不是每一步都同步？当前系统使用 few-step 推理，早期高噪声 step 更影响整体布局、留白、主体轮廓和墨面分布；后期 step 更偏向细节和纹理。很多视频闪烁其实在早期结构判断时就已经产生了。如果只在最后处理，往往已经来不及。所以默认 `key_step_index` 设为 `0`，也就是在最早的关键步做同步。

同步权重不是固定的，而是由光流得到的 `motion_score` 控制：

1. 低运动时，上一帧对齐后的 `x0` 可信度高，可以适当提高同步权重。
2. 中等运动时，权重逐渐下降。
3. 快速运动、遮挡或者光流误差较大时，权重接近 0，避免历史结构拖住当前帧。

当前默认最大同步强度是比较保守的，例如 `strength: 0.18`。这意味着它不是把当前帧替换成上一帧，而是在低运动时引入一小部分历史结构共识。这样做的目标是减少白底呼吸、墨面漂移和轮廓细抖，同时尽量避免拖影。

这一块和 `warped noise` 是互补关系。`warped noise` 让随机扰动更连续；关键步同步让扩散过程中的结构判断更连续。两者都依赖光流，但作用层级不同：一个在噪声层，一个在关键 `x0` latent 层。

## 八、光流带来的稳定性指标：把“看起来抖”变成可比较的数

光流接入后，除了驱动控制模块，还用于建立稳定性指标。这个指标的目的不是替代主观观察，而是提供一个稳定的实验口径，让不同参数、不同模块开关之间可以比较。

指标计算方式是：对连续两帧输入图像估计光流，然后用同一条光流分别计算原画和输出画面的 warp 残差。

原画残差：

```text
e_src = mean(|I_t - warp(I_{t-1}, F)|)
```

输出残差：

```text
e_out = mean(|O_t - warp(O_{t-1}, F)|)
```

这里最关键的是，输出视频也使用输入视频估计出来的同一条光流。这样做的含义是：如果模型输出真的跟随原视频运动，那么上一帧输出经过输入运动对齐后，应该和当前输出接近；如果对齐后仍然差异很大，就说明输出里有额外闪烁、纹理漂移或边缘跳动。

残差会进一步映射成稳定度分数：

```text
s = 100 / (1 + 50 * e)
```

然后计算：

```text
delta = s_out - s_src
```

`delta` 是最常用的观察量。它不是绝对画质分，而是输出相对原视频的稳定性差值。如果 `delta` 更接近 0 或更高，说明输出额外引入的抖动更少；如果 `delta` 很负，说明重绘结果比原视频明显更不稳定。

demo 里还维护了 30 秒滑动平均 `temporal_stability_delta_avg_30s`。这样做是为了避免单帧数值波动太大，更适合看一段视频里的整体趋势。热力图里的分数就是基于这个滑动平均。

这个指标有局限。Farneback 在快运动、遮挡、复杂变形时会不准；水墨视频中有些风格变化本身是可接受的，指标不一定完全等于主观质量。但它至少解决了一个问题：之前只能说“感觉更稳了”，现在可以在同一段素材、同一套参数下比较模块开关带来的趋势。

## 九、主体 Mask 与实时 Shader：重新组织水墨画面的主体、背景和留白

前面几部分主要解决结构和时序问题，但水墨风格还有一个单独的问题：画面应该有取舍。真实水墨画不会把所有区域都用同样强度渲染，通常会突出主体，弱化背景，保留大面积空白。只靠扩散模型和 prompt，很难稳定地区分哪些地方该保留、哪些地方该淡化。

因此我加入了主体 `mask`，并把它接到进 VAE/Latent 前的实时 shader 里。这里的 shader 不是传统图形学里完整的材质系统，而是指一组可解释的图像前处理：降饱和、深度驱动雾化、留白混合、轮廓暗化、模糊和平滑控制。它发生在输入图像进入 VAE 编码之前，所以会影响主图像 latent，但不会污染 `depth` 和 `softedge` 条件本身。

主体 mask 的计算分为两种路径：

1. 如果配置了 SAM2，就在关键帧尝试用 SAM2 提取主体区域。
2. 如果 SAM2 不可用，或者检测失败，就使用 heuristic 回退方案。

heuristic 方案会结合几类信息：深度前景、画面中心先验、光流幅值和 softedge 弱先验。这样即使没有额外分割模型，也能得到一个大致稳定的主体 soft mask。mask 不是硬二值，而是 0 到 1 的 soft alpha，这样后面做雾化和留白时不会产生生硬边界。

为了让 mask 在视频中连续，系统会用光流把上一帧 mask warp 到当前帧，然后和当前帧估计结果做 EMA 混合。运动大时可以降低历史权重，减少拖影；运动小时保留更多历史，减少主体边缘跳动。

mask 接入 shader 的核心位置是雾化和留白控制。旧版前处理主要由 depth 控制：

```text
fog_image = lerp(fog_base, image, pro_depth)
```

它的意思是：前景或近处更多保留原图，远处或背景更多混入白色和模糊，形成水墨里的雾气和留白。

加入主体 mask 后，实际使用的插值权重变成：

```text
fog_lerp = (1 - lambda) * pro_depth + lambda * (1 - subject_mask)
```

其中 `lambda` 对应 `fog_depth_mask_blend`。当 `lambda` 越大，主体 mask 对留白和雾化的影响越强。直观上，主体区域更容易被保留，非主体区域更容易被淡化。这样系统不再只依赖几何深度判断前景背景，而是加入了“主体是否重要”的语义信息。

同时，轮廓暗化仍然使用 `softedge` 控制，不和 mask 混在一起。这样有一个好处：`softedge` 管线条，mask 管主体和背景的保留关系，depth 管空间层次。三者分工比较清楚，后续调参时也容易判断问题来自哪一条支路。

这一部分对毕设展示很关键。因为它说明项目不是简单把视频变黑白，而是在尝试把水墨画面里的主体、留白、雾气和轮廓拆成可控制的部分，再组合进实时扩散链路。

## 十、剩余的时序稳定性增强措施

当前已经完成的模块可以减少一部分闪烁，但系统还没到最终稳定状态。后续我认为主要有几条增强方向。

第一，做 `softedge` 和 `depth` 的时序归一化。现在两类条件图都有逐帧 `min/max` 归一化问题，强度标尺每帧都可能变化。后续可以改成分位数裁剪加 EMA，比如维护一段时间内的条件强度统计，让边缘和深度的动态范围不要每帧重置。这样可以直接针对热力图里发现的 `softedge_scale` 放大抖动问题。

第二，给 `softedge_scale` 加运动感知调度。低运动时，边缘细闪最明显，`softedge` 不一定应该更强；中等运动时可以适当保留轮廓约束；快速运动时又要防止错误边缘拖影。也就是说，`softedge_scale` 不应该永远是固定值，而应该像噪声率和同步权重一样，由运动状态参与控制。

第三，继续调低运动场景的 `noise_rate`。当前代码已经支持 `motion_score -> noise_rate`，但默认配置还没有真正拉开 `min_noise_rate` 和 `max_noise_rate`。后续可以针对静止人像、轻微转头、快速挥手三类素材，分别测试低噪声率对稳定性和画面清晰度的影响。

第四，完善 benchmark。现在热力图已经能看大趋势，但实验流程还需要更干净。每组参数测试前应该显式 reset 后端状态，清空上一帧输入输出、噪声缓存、关键步记忆和稳定度滑动窗口。每组参数至少重复多次，取平均和方差。这样后续论文里引用实验结果会更稳。

第五，考虑更好的光流后端。Farneback 适合实时和快速验证，但在大运动、遮挡、非刚性变形时会不够准。代码里已经预留 `raft` 和 `sea-raft` 后端名。后续如果时间允许，可以接入 SEA-RAFT 这类现成模型，重点比较它对 `warped noise`、mask warp、关键步同步和稳定性指标的一致改善。

第六，评估轻量训练的可能性。当前 `softedge` 分支只是从 `depth` 分支复制初始化，没有针对边缘条件专门适配。如果后续要把 `softedge` 做强，可以考虑只训练很小的 `edge_flow_conv_in` 或轻量 adapter，而不是训练完整 UNet。这样既能降低成本，也能让 edge 分支更适应自己的输入分布。

第七，扩展关键步同步。当前只保存上一帧 `prev_sync_x0`，属于短时递归记忆。后续可以尝试保存更长窗口的关键步记忆，或者只选关键帧作为 anchor。这样可能进一步减少长视频里的风格漂移，但也要控制延迟和显存。

## 十一、向老师汇报时可以这样收束

这次阶段工作的主线可以概括成一句话：我不是只在模型外面加滤镜，而是在实时扩散链路内部逐步加入结构条件、运动分析和时序控制。

`softedge` 的接入解决的是轮廓信息不足的问题；热力图说明了结构条件不是越强越好，尤其高频边缘条件如果没有时序稳定化，反而会放大抖动；光流接入后，系统第一次有了输入运动的显式描述；`warped noise` 和关键步同步分别从噪声层和结构预测层减少帧间随机性；稳定性指标让这些改动可以被量化；主体 mask 和 shader 则把水墨风格里的主体保留、背景淡化、留白控制接入了主图像 latent 之前。

目前项目已经完成了实时主链、双结构条件、光流指标、噪声传播、关键步同步、主体 mask 和 shader 的阶段性实现。下一步不是继续盲目增加模块，而是把实验流程收紧，把 `softedge/depth` 时序归一化、运动感知调度和对照实验做扎实。这样最终论文里可以形成比较完整的逻辑：问题来自哪里，系统怎样改，指标怎样验证，哪些方案有效，哪些方案还有边界。
