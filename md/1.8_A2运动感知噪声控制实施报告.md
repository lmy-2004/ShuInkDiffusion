# A2 运动感知噪声控制：改进与实施报告

## 一、背景与目标

本文档对应 [`1.4_实时水墨时序稳定性提升方案.md`](./1.4_实时水墨时序稳定性提升方案.md) 中的 **A2**：将原先静态的 `do_add_noise` 开关，升级为**随输入视频运动强度可调**的噪声策略，并在工程上引入**基于原视频光流的噪声传播（warped noise reuse）**，以减轻静止、低运动场景下的墨纹闪烁、白底呼吸与轮廓细抖。

目标分层如下。

- **第一层（MVP）**：用轻量光流与 warp 残差得到 `motion_score`，映射为连续 `noise_rate`，在 `encode_image` 与多步去噪 buffer 更新处统一缩放 fresh 噪声强度。
- **第二层（增强）**：在开启时，将上一帧 latent 域噪声沿当前帧估计光流 warp 后与少量新噪声混合，使纹理扰动随内容运动相干，而非每帧完全重采样。

## 二、问题与改造前现状

改造前主要问题（与 1.4 分析一致）包括：

1. **`encode_image` 每帧固定 `randn`**，与 `do_add_noise` 无关，静止输入仍会换一套初始噪声。
2. **`predict_x0_batch` 中 buffer 分支**在 `do_add_noise=True` 时步间使用 `randn_like`，等价于持续注入与运动无关的高频随机性。
3. **Demo 层**虽有 [`1.3`](./1.3_视频时序稳定度指标_调研与实施说明.md) 的 Farneback 光流与 warp 误差，但**仅用于调试指标**，未反馈到采样器。

因此 A2 的落地必须同时触及：**光流计算复用 → 参数下传 → pipeline 内噪声合成逻辑**，缺一不可。

## 三、改进方案概要

### 3.1 运动信号

- 在 [`Live2Diff/demo/temporal_stability.py`](../Live2Diff/demo/temporal_stability.py) 中抽取 **`compute_optical_flow`**，返回 `FlowStats`：`flow`、`map_x`/`map_y`、`e_src`（与既有 warp 残差定义一致）、`mean_flow_mag`。
- 原有 **`compute_flow_warp_errors`** 支持传入已算好的 `flow_stats`，避免同一帧内重复算流。

### 3.2 噪声率映射

- 在 [`Live2Diff/live2diff/pipeline_stream_animation_depth.py`](../Live2Diff/live2diff/pipeline_stream_animation_depth.py) 的 `StreamAnimateDiffusionDepth` 中增加 **`set_noise_control` / `reset_noise_state`**。
- 当 `noise_control_enabled` 且本帧提供 `motion_score`（默认用 `e_src`）时，在 `[low_threshold, high_threshold]` 上线性插值到 `[min_noise_rate, max_noise_rate]`；未启用或首帧无历史时回退为满强度（与原先 `do_add_noise=True` 行为兼容）。
- `do_add_noise=False` 时 `noise_rate` 恒为 0，保持语义。

### 3.3 Warped noise（可选）

- 将输入 RGB 空间的稠密光流双线性缩放到 **latent 网格**，用 `grid_sample` 对上一帧缓存的 **`prev_noise_latent`（encode 路径）** 与 **`prev_buffer_noise`（buffer 路径）** 做 warp。
- 与 fresh 噪声按配置混合：`warped_noise_reuse`、`warped_noise_residual` 与当前 `noise_rate` 共同决定混合比例；高运动场景依赖较高 fresh 比例以降低错误粘连风险（与 1.4 实验矩阵中的风险意识一致）。

### 3.4 生命周期

- **`prepare`（warmup）结束**后调用 **`reset_noise_state`**，避免 warmup 阶段的噪声状态污染在线推理首段。
- Demo 在 **`_reset_temporal_state`** 时同步 **`reset_noise_state`**，与清空上一帧 RGB 缓存一致。

## 五、配置说明（`demo_cfg.yaml`）

当前默认偏向 **稳定优先**（具体数值以仓库内 YAML 为准），典型字段含义如下。

- **`enabled`**：总开关。
- **`low_threshold` / `high_threshold`**：`motion_score`（默认 `e_src`）落在低段→接近 `min_noise_rate`，高段→接近 `max_noise_rate`。
- **`min_noise_rate` / `max_noise_rate`**：噪声强度上下界。
- **`enable_warped_noise`**：是否启用光流 warp 噪声复用。
- **`warped_noise_reuse` / `warped_noise_residual`**：warp 与 fresh 的相对权重相关参数。
- **`flow_max_side`**：与 1.3 一致的 Farneback 计算尺度，控制 CPU 耗时。

调参建议：静止镜头若仍偏闪，可略降 `min_noise_rate` 或 `high_threshold`；快运动若出现拖影或粘连，可提高 `high_threshold`、减小 `warped_noise_reuse` 或临时关闭 `enable_warped_noise`。

## 六、调试与指标

- **原有**：`temporal_stability_*`、`e_src` / `e_out` 等仍由 `vid2vid` 维护。
- **新增（经 `get_debug_info`）**：`noise_control_enabled`、`motion_score`、`mean_flow_mag`、`noise_rate`、`warped_noise_enabled`、`warped_noise_reuse`、`warped_noise_residual`。

便于对照「同一帧：运动估计 → 噪声率 → 主观闪烁」是否一致。

## 七、与外部方法的对应关系（简述）

- **StreamDiffusionV2** 类思路：用输入运动信息调节噪声强度，本实现采用 **warp 残差 `e_src` + 线性映射**，不做完整系统复刻。
- **Go-with-the-Flow**：强调光流相关噪声与时空相干；本实现为 **推理侧工程近似**（`grid_sample` + 残差 fresh），未实现论文中的严格 Gaussianity 保持算法，以降低接入成本。

## 八、已知局限与后续工作

1. **运动源单一**：当前默认以 `e_src` 为主信号；`mean_flow_mag` 已传入 pipeline，可按需在映射中组合或切换（计划中的 `motion_source` 可后续显式化到 YAML）。
2. **Warmup 内多步加噪**仍为原逻辑（`randn_like`），与 1.4 计划「首轮不改 warmup」一致；若需全链一致可再开子任务。
3. **Farneback** 在快运动、遮挡处误差大，阈值需结合画面解读；更重光流可换模型但会抬成本。
4. 与 **A3（depth/softedge 时序归一化）**、**softedge 运动调度** 并列时，建议分阶段做 A/B，避免多因子的同时改动难以归因。

## 九、验证建议（与计划一致）

建议至少三类素材回归：

1. **静止镜头**：观察 `noise_rate` 是否明显偏低、白底与细轮廓是否更稳。
2. **缓慢运动**：观察 warped noise 是否增强连贯且不过度「糊死」笔触。
3. **快速运动**：观察是否自动抬高 fresh 比例、有无纹理粘连或拖影。

静态检查：`py_compile` 已通过；完整视频流与主观评测需在目标机器与真实相机/片源上执行。
