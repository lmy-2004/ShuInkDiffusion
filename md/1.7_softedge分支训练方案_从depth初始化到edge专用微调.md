# softedge 分支训练方案：从 depth 初始化到 edge 专用微调

## 目标

本文回答一个非常具体的问题：

> 当前 `edge_flow_conv_in` 只是从 `flow_conv_in` 复制初始权重。  
> 如果要把它真正训练成适配 `softedge` 的专用分支，应该怎么做？

本文的输出不是泛泛而谈，而是给出一套尽量贴合当前 `Ink-Diffusion / Live2Diff` 工程现状的训练方案，包括：

1. 为什么“先复制 depth 权重，再微调成 edge 专用”是当前最稳的路线。
2. 应该训练哪些参数，冻结哪些参数。
3. 数据应该怎样组织。
4. loss 应该怎么设计。
5. 如何分阶段训练。
6. 如何评估它到底有没有把 `softedge` 从“轮廓增强器”变成“轮廓稳定器”。

---

## 一、结论先行

如果目标是最小改动、最高成功率，那么当前最合理的方案不是：

- 直接给 `edge_flow_conv_in` 一套纯随机权重
- 也不是继续长期使用“从 depth 拷过来但不训练”的权重

而是：

1. 保留当前 `flow_conv_in -> edge_flow_conv_in` 的复制初始化。
2. 冻结大部分主干。
3. 先只训练 `edge_flow_conv_in`。
4. 再视效果决定是否少量放开 `conv_in` 后的局部层或 LoRA。
5. 用“单帧生成质量 + 短时时序一致性 + 水墨轮廓稳定性”联合约束。

这条路线的本质，是把 `edge` 分支当成一个**条件适配器**来训，而不是把整个视频扩散模型重新训一遍。

更直白地说：

- `depth` 分支已经学会了“怎样把结构条件翻译成 UNet 能听懂的偏置”
- `edge` 分支需要做的，不是从零学会整个风格生成
- 而是学会“怎样把高频 edge 条件翻译成**不会引起闪烁**的结构偏置”

所以它更像一个**受控微调问题**，而不是全模型重训问题。

---

## 二、为什么这条路线最合理

### 1. 当前 `edge` 分支已经有一条可工作的通路

当前 UNet 的入口注入逻辑是：

- 主图像 latent 先经过 `conv_in`
- `depth_sample` 经过 `flow_conv_in`
- `softedge_sample` 经过 `edge_flow_conv_in`
- 最后和主干特征直接相加

这意味着：

- `edge` 分支的架构位置已经对了
- 不需要再发明新接口
- 真正缺的是“参数适配”

所以当前最该训练的不是“有没有这条分支”，而是“这条分支的参数是否真的懂 edge 信号”。

### 2. `flow_conv_in` 提供的是很好的 warm start

`flow_conv_in` 已经随 `live2diff.ckpt` 一起训练过，至少学会了几件有价值的事：

- 如何把条件 latent 映射到 UNet 输入特征空间
- 如何让条件分支和主干特征相加而不立刻炸掉
- 如何在当前主干统计范围内输出合理幅值

这几件事对 `edge` 分支也重要。

所以复制初始化的价值不在于“depth 权重等于 edge 权重”，而在于：

> 它给 `edge` 分支提供了一个位于正确数值范围、正确特征空间、正确接口语义上的起点。

这和 ControlNet 的思路很接近：

- 尽量复用已有强 backbone
- 新控制分支不要从完全无意义的状态起跑
- 先保证不会破坏原模型，再逐步学条件

外部资料也支持这点。  
[ControlNet 论文](https://huggingface.co/papers/2302.05543) 与其训练文档/教程都强调：

- 锁住大模型 backbone
- 新控制路径从安全初值出发
- 用小规模可训练模块去学新条件

这和当前项目很契合。

### 3. 纯随机初始化在你这个项目里风险更高

如果 `edge_flow_conv_in` 用随机权重直接训或直接推理，当前架构下会有三个现实问题：

1. 它不是挂在末端，而是直接加到 UNet 入口特征上。
2. `softedge` 是高频、稀疏、动态范围敏感的条件。
3. 当前推理是视频流式，不是单张图容错。

所以随机初始化的副作用很可能表现为：

- 训练前期 loss 波动更大
- 很容易学成“过度锐化”或“边缘闪烁放大”
- 甚至在验证时出现稳定性比 depth-only 还差的情况

相比之下，复制 depth 权重至少能先让它处在一个“不会完全乱输出”的状态。

## 三、外部研究对这条路线的启发

这里不做大而全综述，只保留对当前训练设计最直接有用的结论。

### 1. ControlNet：冻结 backbone，让新条件分支从安全初值起跑

[Train your ControlNet with diffusers](https://huggingface.co/blog/train-your-controlnet) 和 [ControlNet 项目](https://github.com/lllyasviel/ControlNet) 传递了三个核心经验：

1. 条件控制训练不一定要改整个主干。
2. 新条件分支适合单独训练。
3. 小数据下，冻结 backbone 往往比全模型重训更稳。

虽然你这里不是标准 ControlNet residual 路线，但思路是共通的：

- 主干是已经可用的大模型
- `edge_flow_conv_in` 是新条件适配器
- 目标不是重建整个模型能力，而是让新条件变得可控、可用、稳定

### 2. Synchronized Multi-Frame Diffusion：时序一致性要尽早进入约束

[Highly Detailed and Temporal Consistent Video Stylization via Synchronized Multi-Frame Diffusion](https://arxiv.org/abs/2311.14343) 指出：

- 视频一致性不能只在最后看结果
- 更有效的是让多帧在去噪早期就共享结构与颜色共识

对当前项目的启发是：

- `edge` 分支训练不能只盯单帧生成质量
- 训练阶段就应该让相邻帧的一致性参与监督

否则很容易学成“单帧轮廓更强，但视频更闪”。

### 3. ReCoNet / FDB：时序 loss 不一定非得很重，但必须存在

[ReCoNet](https://arxiv.org/abs/1807.01197) 强调：

- 输出层 warping consistency
- feature map 级 temporal loss

[Frame Difference-Based Temporal Loss for Video Stylization](https://arxiv.org/abs/2102.05822) 则说明：

- 即使不依赖重型光流，也可以用 frame-difference loss 获得接近 OFB loss 的效果

对你当前项目的意义非常明确：

- 可以先上轻量时序 loss 作为第一阶段
- 再用光流 warp loss 做第二阶段强化

这样训练门槛更低，也更适合先验证 `edge` 分支是否学到了稳定控制。

### 4. Go-with-the-Flow：随机噪声本身也是时序稳定的一部分

[Go-with-the-Flow](https://arxiv.org/abs/2501.08331) 强调：

- 仅靠调整条件分支还不够
- 噪声构造也会直接影响视频一致性

对本项目的启发是：

- `edge` 分支训练期间，不要忽略噪声策略
- 至少在训练后期，应该让训练时的噪声设置尽量接近未来稳定模式

否则会出现：

- 训练时 `edge` 分支在高 fresh-noise 条件下学会了一套补偿
- 推理时你又改成低噪声稳定模式
- 分支统计再次失配

## 四、当前项目最现实的训练目标是什么

### 不建议把目标设成“让 edge 更强”

这是一个非常容易走偏的点。

如果目标写成：

- 边更清楚
- 轮廓更重
- 结构更明显

那训练很可能学出一个“边缘增强器”。  
它在单帧上看起来更有控制力，但在视频里更容易闪。

### 更合理的目标应该是：

> 在不破坏当前 Live2Diff 主体风格能力的前提下，让 `edge_flow_conv_in` 学会把 `softedge latent` 变成**时序稳定、幅值受控、对轮廓有帮助**的结构偏置。

这句话可以拆成 4 个约束目标：

1. **条件有效**  
   `softedge` 真的参与控制，而不是被训练成几乎没影响。

2. **幅值受控**  
   不允许 `edge` 分支输出轻易压过主干或 depth 分支。

3. **时序稳定**  
   相邻帧的轮廓与边界不应因 `edge` 分支而额外变抖。

4. **水墨友好**  
   关注的不是普通纹理锐化，而是水墨轮廓、留白边界、主体边缘稳定。

---

## 五、推荐的总体训练路线

我建议分三阶段。

---

## 六、阶段 A：只训练 `edge_flow_conv_in`

### 目标

先验证一件最重要的事：

> 在冻结主干的情况下，`edge_flow_conv_in` 本身能不能学会“稳定地使用 softedge 条件”。

这是成本最低、归因最清晰的一步。

### 冻结策略

冻结：

- `conv_in`
- 所有 down / mid / up blocks
- temporal attention / motion modules
- text encoder
- VAE
- `flow_conv_in`

只训练：

- streaming UNet 的 `edge_flow_conv_in.*`
- warmup UNet 的 `edge_flow_conv_in.*`

必要时可以把 warmup / streaming 的 `edge_flow_conv_in` 设成共享参数，先简化问题；如果实现上不方便，就各自训练，但初始化保持一致。

### 为什么先这么做

因为这一阶段的目标不是追最强效果，而是验证：

- `edge` 分支有没有独立学习空间
- 会不会一训练就塌
- 是否存在明显比复制不训练更好的趋势

如果连这一阶段都没有收益，那么继续放开主干只会增加变量，无法说明问题出在哪。

---

## 七、阶段 B：在 `edge_flow_conv_in` 基础上做轻量联训

如果阶段 A 证明 edge 分支有效，再进入阶段 B。

### 训练对象

继续训练：

- `edge_flow_conv_in`

额外放开少量参数，二选一：

1. **最稳方案**：只给 `conv_in` 或注入邻近层加 LoRA
2. **稍激进方案**：放开 `conv_in` 和第一层 down block 的少量参数

### 为什么不建议直接放开大部分 UNet

因为那样很容易发生：

- 主干替 edge 分支“背锅”
- 你以为是 edge 学会了，其实是主干整体偏了
- 最终单帧更好，但 prompt / 风格 / 时序行为都一起变

对于当前项目，“可归因”比“一次性极限提分”更重要。

---

## 八、阶段 C：再考虑噪声与时序机制协同微调

只有在阶段 A/B 都已经证明 edge 分支可学之后，才建议做这一阶段。

这一阶段可以考虑：

1. 训练时启用更接近稳定模式的 noise control
2. 训练时加入 warped noise / low-motion noise schedule
3. 必要时少量联训与时序相关的轻量层

这一阶段不是必需，但如果最终目标是视频展示级稳定性，它很可能会带来边际收益。

---

## 九、数据应该怎么准备

这是最关键的现实问题之一。

### 理想数据

最理想的数据形式是：

- 输入视频片段 `x_t`
- 对应目标水墨视频片段 `y_t`
- 对每帧配套 prompt / style prompt

然后从输入帧在线生成：

- `depth_map`
- `softedge_map`

这样训练时，模型学的是：

- 输入视频 + prompt + 条件图
- 去拟合目标水墨视频

### 如果没有成对水墨视频

那么建议分两档。

#### 档 1：伪配对蒸馏

做法：

1. 用当前系统离线生成一批质量较好的水墨视频
2. 人工筛掉明显闪烁、崩坏样本
3. 把剩下样本当作 teacher target
4. 用它们训练 `edge` 分支，让 student 学会更稳定地复现 teacher 风格

这不是最终最优，但适合冷启动。

优点：

- 快
- 数据容易造
- 先验证 edge 分支能否学会稳定控制

风险：

- teacher 本身上限有限
- 容易把 teacher 的缺点一起蒸馏下来

#### 档 2：少量人工精选成对视频

如果你手头有少量高质量样本，建议优先做“小而干净”的训练集，而不是一开始追大数据。

对当前任务，比起数量，更重要的是覆盖下面三类场景：

1. 静止镜头  
   最能暴露轮廓细闪和留白抖动。

2. 低速运动镜头  
   最适合训练“边缘该跟着走，但不要抖”。

3. 中速运动镜头  
   用来避免学成过强粘连或轮廓拖影。

### 数据组织建议

每个样本建议是一个短 clip，而不是完全独立帧。

推荐：

- clip 长度：`8 ~ 16` 帧
- 分辨率：先 `256` 或 `384` 训练，后续再升
- 帧率：优先保持真实时序，不要过度抽帧

每个 clip 建议保存：

- `src_frames`
- `tgt_frames`
- `prompt`
- 可选：`negative_prompt`
- 可选：预计算 `flow / occlusion mask`

## 十、训练目标怎么定义

这部分决定“学成边缘增强器”还是“学成稳定控制器”。

我建议 loss 分三层：

1. 基础生成重建层
2. 时序稳定层
3. edge 分支约束层

---

## 十一、基础生成层

### 1. 扩散主损失

如果沿用扩散训练范式，主损失仍然应该是标准噪声预测/去噪损失。

如果你使用目标视频 latent 做监督，那么主损失可以保持当前扩散范式不变。  
这部分不需要为了 edge 分支专门发明新东西。

### 2. 感知 / 重建辅助损失

为了避免只靠扩散损失导致收敛慢、边缘学不明白，建议加轻量辅助项：

- `L1` 或 `Charbonnier` 输出重建损失
- LPIPS 感知损失
- 可选：VGG 特征损失

作用：

- 帮助早期稳定训练
- 让 edge 分支更快学会“轮廓对目标外观有帮助”

---

## 十二、时序稳定层

这部分是本方案和普通条件微调最不一样的地方。

### 方案 A：先上轻量 FDB temporal loss

参考 [Frame Difference-Based Temporal Loss for Video Stylization](https://arxiv.org/abs/2102.05822)，第一阶段推荐先用 frame difference loss：

`L_fdb = || (O_t - O_{t-1}) - (Y_t - Y_{t-1}) ||`

可以同时做：

- pixel 空间
- feature 空间

它的优点是：

- 不依赖高质量光流
- 实现简单
- 很适合先把“抖动幅度”压下来

### 方案 B：再加 flow warping temporal loss

在阶段 B 或 C，再引入光流 warp loss：

- 用输入视频估计 `F_{t-1->t}`
- 把上一帧输出 `O_{t-1}` warp 到当前帧
- 与当前输出 `O_t` 做差
- 遮挡区可减权或掩蔽

形式上类似：

`L_warp = || O_t - warp(O_{t-1}, F) || * M`

其中：

- `F` 来自输入视频光流
- `M` 是 occlusion / confidence mask

这个 loss 和你现在 benchmark 指标的方向是一致的，所以训练目标和评估目标会更对齐。

### 方案 C：feature map temporal loss

参考 [ReCoNet](https://arxiv.org/abs/1807.01197)，可以在中间特征上加一层轻量 temporal consistency：

- 取 `conv_in` 后或 early block 的 feature
- 对相邻帧做 warp
- 惩罚同一物体特征在 traceable 区域的无意义波动

这个损失的价值在于：

- 它比最终 RGB 层更早约束 edge 分支
- 能直接告诉模型“不要让边缘条件把中间结构特征搞抖”

### 推荐顺序

不建议一开始就全开。

建议：

1. 阶段 A：`L_diffusion + L_recon + L_fdb`
2. 阶段 B：加入 `L_warp`
3. 阶段 C：如有必要再加 `L_feat_temporal`

这样更稳，也更容易定位收益来自哪里。

---

## 十三、edge 分支专属约束

这一层很重要，否则 edge 分支容易学成“输出幅值越来越大”。

### 1. 输出幅值正则

建议对 `edge_flow_conv_in(condition)` 的输出加轻量幅值约束：

- `L_edge_amp = ||E||_1` 或 `||E||_2`

目的不是把它压成 0，而是防止它靠无脑放大来抢控制权。

### 2. 帧间 edge embedding 平滑

可以直接约束 edge 分支输出 embedding 的相邻帧差异：

`L_edge_smooth = || E_t - warp(E_{t-1}, F) ||`

其中：

- `E_t = edge_flow_conv_in(softedge_t)`

这个 loss 的意义非常直接：

- `softedge` 输入本身可能抖
- 但 edge 分支输出不应该同样抖

也就是说，它在训练 edge 分支学会“过滤掉不稳定的高频波动”。

### 3. 与 depth 分支的相对幅值约束

当前主链是 `sample + depth_embed + edge_embed`。  
如果 edge 输出长期远大于 depth 输出，就很容易把系统推向“边缘过驱动”。

因此建议监控：

- `mean(|edge_embed|)`
- `mean(|depth_embed|)`
- 二者比值

训练时可加软约束，让边缘分支不要长期压倒 depth 分支。

---

## 十四、推荐 loss 组合

推荐第一版总损失：

`L = L_diff + λ_rec L_rec + λ_fdb L_fdb + λ_amp L_edge_amp + λ_es L_edge_smooth`

推荐初始权重思路：

- `λ_rec`: 中等
- `λ_fdb`: 中等偏低，先稳住趋势
- `λ_amp`: 小
- `λ_es`: 小到中等

加入光流后可变为：

`L = L_diff + λ_rec L_rec + λ_fdb L_fdb + λ_warp L_warp + λ_amp L_edge_amp + λ_es L_edge_smooth`

不建议一开始把 `λ_warp` 设很高。  
因为训练初期输出质量不稳，光流 loss 容易把模型拉向过平滑。

---

## 十五、训练时的输入与采样策略

### 1. 输入 clip 采样

建议一个 batch 中混合三类 clip：

1. 低运动 clip
2. 中运动 clip
3. 轮廓复杂 clip

原因：

- 低运动 clip 负责训练“不要闪”
- 中运动 clip 负责训练“该跟着动的时候要能动”
- 轮廓复杂 clip 负责训练“边缘条件不能失真”

### 2. `softedge_scale` 不要固定只训一个值

建议训练时对 `softedge_scale` 做随机采样，例如：

- 主要采样 `0.0 ~ 0.4`
- 少量采样 `0.4 ~ 0.8`

理由：

- 未来推理时分支要适应不同强度
- 如果永远只在 `0.6` 训练，分支很可能只会在那个点附近有效

### 3. 训练时 noise control 要逐步靠近稳定模式

建议：

- 阶段 A 先保持接近当前默认
- 阶段 B 开始降低低运动样本的 fresh noise rate
- 阶段 C 可以加入 warped noise 训练

这样能减少训练-推理统计错位。

## 十六、超参数建议

这部分只给第一版建议，不假装是唯一答案。

### 优化器

- `AdamW`
- 或显存紧张时 `8-bit Adam`

### 学习率

#### 阶段 A：只训 edge 分支

- `1e-4` 起步可尝试
- 保守一点可用 `5e-5`

因为只训小分支，学习率可以比全模型微调略大。

#### 阶段 B：轻量联训

- `edge_flow_conv_in`: `5e-5`
- 新放开的 LoRA / 小层：`1e-5 ~ 2e-5`

### batch

如果显存允许：

- clip batch size `2 ~ 4`

如果显存紧张：

- batch size `1`
- 用 gradient accumulation 凑有效 batch

### 训练步数

第一阶段不建议上来训很久。

建议：

- 先做 `10k ~ 30k` steps 验证趋势
- 中间固定验证集高频看图和看时序指标

原因很简单：

- 当前任务最怕学偏
- 一旦过拟合，你会看到轮廓变得“更重但更假”

---

## 十七、验证集应该怎么构造

验证集不应只看随机视频。

建议至少拆三组：

### 1. 静止脸部 / 低运动人像

看：

- 眼睛边缘
- 鼻梁线
- 嘴角轮廓
- 脸与背景交界

这是 `softedge` 最容易帮忙，也最容易坏事的区域。

### 2. 缓慢转头 / 手部小运动

看：

- 轮廓是否跟得上运动
- 有没有边缘粘连
- 有没有线条粗细跳动

### 3. 背景留白明显的镜头

看：

- 背景边缘是否更稳
- 留白是否更干净
- 是否出现背景脏纹理增加

---

## 十八、评估指标怎么定

建议不要只看视觉图，也不要只看单一数值。

### 自动指标

保留当前已有：

- `temporal_stability_delta`
- `temporal_stability_delta_avg_30s`

新增建议：

1. `edge_region_warp_error`  
   只在边缘区域统计 warp error，更贴近本任务。

2. `background_whitespace_stability`  
   对低纹理/高亮背景区域统计时序波动。

3. `edge_embed_magnitude_stats`  
   跟踪 `edge` 分支输出均值、方差、与 `depth` 分支比值。

### 人工评测

每轮至少做三项主观判断：

1. 轮廓稳定度
2. 留白稳定度
3. 水墨自然度

因为只看稳定性数值，模型很容易走向：

- 更稳
- 但更糊
- 或者更“电子描边”

---

## 十九、推荐的消融实验

这一部分一定要做，否则训练结果很难解释。

建议至少比较下面几组：

1. `depth-only`
2. `depth + softedge(copy only, no train)`
3. `depth + softedge(train edge branch only)`
4. `depth + softedge(train edge branch + FDB loss)`
5. `depth + softedge(train edge branch + warp loss)`
6. `depth + softedge(train edge branch + light joint tuning)`

这样你才能回答三个问题：

1. `edge` 分支是否真的可学
2. 时序 loss 是否真的有必要
3. 是否值得继续放开更多参数

---

## 二十、落地到当前仓库，最小实现应该怎么做

由于当前仓库基本没有现成训练器，建议新增一套最小训练脚手架，而不是强行把训练逻辑塞进 demo 推理代码。

### 建议新增内容

1. `train_softedge_branch.py`
2. `datasets/video_pair_dataset.py`
3. `losses/temporal_losses.py`
4. `configs/train_softedge_branch.yaml`

### 核心职责

`train_softedge_branch.py`

- 加载当前 `Live2Diff` checkpoint
- 执行 `init_softedge_from_depth()`
- 冻结主干
- 构造 clip 训练循环
- 记录验证图和时序指标

`video_pair_dataset.py`

- 读短视频 clip
- 返回 `src_frames / tgt_frames / prompt`
- 在线或离线生成 `softedge`

`temporal_losses.py`

- `frame difference loss`
- `warp loss`
- `edge embedding smooth loss`

### 实现顺序

建议按这个顺序，不要一开始做太全：

1. 只训 `edge_flow_conv_in`
2. 只加 `recon + FDB`
3. 验证有效后再上 `warp loss`
4. 最后再考虑 LoRA / 轻量联训

---

## 二十一、为什么我不推荐第一版就直接做完整 ControlNet-lite

从研究角度，ControlNet-lite residual 路线是合理的。  
但从当前工程投入产出比看，第一版不建议直接上。

原因：

1. 你现在已经有 `softedge_sample -> edge_flow_conv_in -> sample + cond` 这条通路
2. 当前核心问题首先是“参数适配”和“时序稳定监督”
3. 如果连这条更小的路都训不出来，直接上更复杂的 residual 架构只会增加变量

所以更稳妥的策略是：

- 先把现有分支训对
- 再决定是否值得进一步升级架构

---

## 二十二、风险与常见失败模式

### 1. 学成“描边器”

表现：

- 单帧轮廓更明显
- 视频里边缘更闪

原因：

- 只做重建，不做时序约束

### 2. 学成“抹平器”

表现：

- 稳定性数值上去了
- 但轮廓软塌、笔触没了

原因：

- `warp loss` 权重太高
- 对 edge 分支输出约束过强

### 3. 主干替 edge 分支背锅

表现：

- 看起来效果变好
- 但 prompt、风格、整体行为一起漂

原因：

- 放开了太多主干参数

### 4. 训练-推理统计错位

表现：

- 训练验证好看
- demo 实时跑起来却没提升或更差

原因：

- 训练时 clip、噪声、归一化、scale 分布和推理时不同

---

## 二十三、我建议的第一版方案

如果只允许做一版最小可行训练，我建议是下面这套：

### 第一版 MVP

1. 初始化  
   `edge_flow_conv_in <- flow_conv_in`

2. 冻结  
   除 `edge_flow_conv_in` 外全部冻结

3. 数据  
   用短 clip 成对视频，优先低运动和缓慢运动场景

4. loss  
   `L_diff + L_rec + L_fdb + L_edge_amp`

5. 训练步数  
   先 `10k ~ 20k`

6. 验证  
   固定 3 组视频  
   看 `delta_avg_30s`、边缘区域 warp error、主观视频对比

### 为什么是这版

因为它最容易回答：

> `edge_flow_conv_in` 单独训练，能不能把 softedge 从副作用源变成收益源？

如果这一步答案是“能”，再继续上：

- `warp loss`
- `edge embedding smooth`
- 轻量联训

如果这一步答案是“不能”，那说明问题不只是参数，而是架构或条件统计本身也要改。

---

## 二十四、第二版增强方案

如果 MVP 成功，第二版建议：

1. 加入 `warp loss`
2. 加入 `edge embedding smooth`
3. 低运动样本降低训练时 fresh noise rate
4. 对 `conv_in` 或第一层 down block 加 LoRA 联训

第二版目标不是单纯“继续提稳定性”，而是追求：

- 轮廓稳定
- 不失笔触
- 不明显变糊

---

## 二十五、最终建议

对于当前项目，我的整体判断是：

### 1. 值得训练，而且优先级不低

因为当前 `softedge` 的核心问题已经不是“有没有接进去”，而是：

- 接进去了
- 但统计不匹配
- 也没有经过 edge 专属训练

这正是最适合做小范围条件分支微调的场景。

### 2. 最稳的路线就是“复制 depth，再训成 edge”

这条路线同时满足：

- 工程改动小
- 风险可控
- 容易归因
- 与外部成熟方法一致

### 3. 训练重点不要放在“更强边缘”，而要放在“更稳边缘”

如果目标错了，模型很可能学会：

- 轮廓更重
- 但视频更闪

如果目标对了，模型应该学会：

- 不把每帧 `softedge` 的高频抖动原样放大
- 而是提取其中对结构真正有用、且对时序更友好的部分
