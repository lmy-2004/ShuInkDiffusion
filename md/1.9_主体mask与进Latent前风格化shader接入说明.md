# 主体 mask 与进 Latent 前风格化（实时 shader）接入说明

本文说明 **主体 mask 如何计算**、**如何进入项目管线**，以及 **在进 VAE/Latent 前的风格化里如何用 depth 与 mask 共同驱动留白/雾气 lerp**。

---

## 一、代码位置与调用顺序

核心逻辑在 [`Live2Diff/live2diff/pipeline_stream_animation_depth.py`](../Live2Diff/live2diff/pipeline_stream_animation_depth.py) 的 `__call__` 与 `build_stylized_image`。

单帧推理的大致顺序为：

1. `build_depth_map`：得到归一化深度图（供 UNet 条件与风格化）。
2. 若需要：`build_softedge_map`（软描边条件 + 风格化轮廓项）。
3. 若需要：`build_subject_mask_map`（主体 soft mask，单通道 0–1）。
4. 若开启 `enable_stylize_preprocess` 且存在 `softedge_map`：`build_stylized_image`（进 latent 前的「shader」）。
5. `encode_image` → UNet → `decode_image`。

Demo 侧参数由 [`Live2Diff/demo/vid2vid.py`](../Live2Diff/demo/vid2vid.py) 的 `InputParams` 与 YAML 注入；配置文件示例见 [`Live2Diff/demo/demo_cfg.yaml`](../Live2Diff/demo/demo_cfg.yaml) 中的 `subject_mask` 与 `stylization_kwargs`。

## 二、主体 mask 如何计算

`build_subject_mask_map(image, depth_map, softedge_map)` 输出 **与帧同分辨率的单通道 alpha**（内部 0–1，Debug 预览为灰度图）。

### 2.1 何时计算

`_needs_subject_mask_map()` 为真当：

- `subject_mask_debug == true`（便于 Debug 预览，即使未开进 latent 前风格化也会算 mask），或
- `enable_stylize_preprocess == true`（只要开启进 latent 前风格化，**一定会**算主体 mask，并参与下文雾/留白 lerp；无单独开关）。

### 2.2 后端与关键帧

- `subject_mask_requested_backend`：`auto` | `sam2` | `heuristic`。
- **关键帧**：每隔 `subject_mask_keyframe_interval` 帧，在 `auto`/`sam2` 下尝试 SAM2；失败或未配置则回退 **heuristic**。
- **SAM2**（可选）：需 `sam2` 包及 `pipe.sam2_model_cfg` / `pipe.sam2_checkpoint`，或环境变量 `SAM2_MODEL_CFG`、`SAM2_CHECKPOINT`。候选 mask 按中心先验、面积、预测 IoU 等打分选一块区域。
- **Heuristic**（默认保底）：融合
  - 深度相对前景（中心 vs 边缘深度）、
  - 画面中心高斯先验、
  - 光流幅值（若有）、
  - 软描边弱先验（若有），  
  再经 smoothstep、形态学开闭、高斯平滑，得到较稳的 soft mask。

### 2.3 时序连续性

- 使用与噪声控制同源的光流场，对上一帧 mask 做 **warp**，再与当前估计 **EMA 混合**（运动大时略降 EMA，减轻拖影）。
- 帧计数 `subject_mask_frame_idx` 在 `reset_noise_state` 时清零，与会话重置一致。

## 三、进 Latent 前风格化里 mask 如何接入（雾 / 留白 lerp）

原先在 `build_stylized_image` 中，经处理后的深度 `pro_depth ∈ [0,1]^3` 控制：

\[
\texttt{fog\_image} = \texttt{lerp}(\texttt{fog\_base},\ \texttt{image},\ \texttt{pro\_depth})
\]

即：**深度大（近/前景）处更多保留原图，深度小处更多雾与白**。

### 3.1 depth 与 mask 的线性混合

当 `build_stylized_image` 收到非空的 `subject_mask_01` 时（在开启 `enable_stylize_preprocess` 且走完 `build_subject_mask_map` 后通常总有），会先做 `pro_mask = 1 - mask`，再做轻度高斯平滑（`subject_mask_fog_blur_sigma`），随后定义：

\[
\texttt{fog\_lerp} = (1 - \lambda)\,\texttt{pro\_depth} + \lambda\,\texttt{pro\_mask},
\quad \lambda = \texttt{fog\_depth\_mask\_blend} \in [0,1]
\]

最终：

\[
\texttt{fog\_image} = \texttt{lerp}(\texttt{fog\_base},\ \texttt{image},\ \texttt{fog\_lerp})
\]

语义上：

- \(\lambda = 0\)：与旧版一致，仅 depth 驱动留白。
- \(\lambda = 1\)：完全由**反相后的**主体 mask 驱动（非主体区域权重更高，更易雾化/留白）。
- 中间值：在「几何深度」与「语义主体」之间折中，适合风景里 depth 不明确但希望压住背景纹理的场景。

轮廓暗化分支仍使用 `pro_outline`（softedge），与 mask 解耦，避免边缘与留白控制混成一团。

### 3.2 调试缓存

- `last_processed_depth_map`：仍为纯 depth 支路后的 `pro_depth`（便于对照）。
- `last_processed_fog_lerp_map`：实际用于 `fog_image` 的 `fog_lerp`（depth/mask 混合后）。

## 四、配置项（YAML / Demo UI）

在 `demo_cfg.yaml` 的 `subject_mask` 下典型字段：

| 键 | 含义 |
|----|------|
| `debug` | 是否在**未**开启风格化时也计算 mask（仅 Debug 预览）。 |
| `fog_depth_mask_blend` | \(\lambda\)，depth 与 mask 插值权重（风格化开启时始终参与雾 lerp）。 |
| `fog_blur_sigma` | mask 参与雾 lerp 前的平滑（与 depth 的 blur 独立）。 |
| `backend` / `keyframe_interval` / `ema` | mask 后端与稳定性参数。 |

另：`stylization_kwargs.enable_stylize_preprocess: true` 时一定会算 mask 并写入雾 lerp。Demo 前端对应 `InputParams`：`fog_depth_mask_blend`、`subject_mask_fog_blur_sigma`。

---

## 五、与 UNet 条件的关系

当前 **mask 不直接作为第三路 UNet 条件**；它只影响 **VAE 编码之前的 RGB 风格化**（与文档中「进 latent 前风格化」一致）。后续若要把 mask 编码进 latent，可沿用现有 `last_subject_mask_map` 与 `_encode_condition_map` 模式扩展，本文不展开。

---

## 六、小结

- **Mask**：关键帧 SAM2 + 回退 heuristic + 光流 warp + EMA，输出 soft alpha。
- **接入 stylize**：用 `fog_lerp = lerp(pro_depth, pro_mask, λ)` 替换原先单独的 `pro_depth` 驱动 `fog_image`，实现 depth 与主体 mask 共同的实时留白/雾气控制。
